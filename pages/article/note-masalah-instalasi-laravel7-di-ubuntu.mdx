Ini merupakan catatan saya ketika menginstall laravel di Ubuntu 18.04 dan mendapati beberapa pesan error. Sejauh ini adal 4 pesan error yang saya alami dan ngga menutup kemungkinan suatu saat bakal ada error-error lagi yang bermunculan atau bahkan error ini muncul kembali.

> Note: Saya tidak akan membahas bagaimana pesan error ini bisa terjadi atau membahas lebih rinci, akan tetapi saya akan membahasnya dengan metode penyelesaian masalah.

## Inti Permasalahan

Berikut beberapa daftar pesan error yang saya alami, mulai dari konfigurasi composer secara global di sistem operasi sampai ke konfigurasi laravel.

 1. `zsh: laravel command not found`
 2. `php artisan serve warning require (../vendor/autoload.php): failed to open stream: no such file or directory in ../artsan`
 3. `error: Your requirements could not be resolved to an installable set of packages`
 4. `laravel 500 internal server error`

## Pemecahan Masalah

Silahkan buka terminal kamu dan masukkan perintah berikut sesuai dengan nomor pesan error di bagian Inti Permasalahan.

 1. `composer global about` dan ada muncul pesan `"Changed current directory to /home/username/.config/composer"` untuk mengatasinya silahkan buka file konfigurasi zshrc-nya dengan perintah `sudo nano ~/.zshrc` dan tambahkan ini `alias laravel=".config/composer/vendor/bin/laravel"` di bagian paling bawah.
 2. `composer instll` or  `composer update`
 3. `composer install --ignore-platform-reqs` or `composer update --ignore-platform-reqs`
 4. Pertama masuk ke direktori projek laravel kamu dan ubah file `.env-example` menjadi `.env` lalu jalankan perintah `php artisan key:generate` setelah itu bersihkan cache dengan perintah `php artisan cache:clear && php artisan config:clear`
